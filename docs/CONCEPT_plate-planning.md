# Концепт: Планирование через "Сборку тарелки"

## Одно предложение

Игрок собирает 3 тарелки (завтрак, обед, ужин) из ограниченного набора продуктов, а живой BG-график показывает последствия каждого решения в реальном времени.

---

## Ядро: два фактора вместо шести

Текущая фаза планирования требует одновременного учёта ~6 ограничений (WP, углеводы×3 сегмента, размеры слотов, блокировки, правила упражнений, инвентарь). Новая механика сводит всё к **двум осям напряжения:**

- **Сытость** — нужно набрать достаточно (нельзя не есть)
- **BG-кривая** — нужно удержать в зелёной зоне (нельзя переесть)

Конфликт между этими двумя осями — единственное, что игрок держит в голове. Вся глубина возникает из того, *как* он разрешает этот конфликт.

---

## Почему это даёт интерес

### 1. Нелинейные синергии — "2+2 ≠ 4"

У каждого продукта есть теги (Carb, Protein, Fiber, Fat, Sugar). Когда продукты оказываются на одной тарелке, их теги **взаимодействуют** — BG-эффект тарелки не равен сумме эффектов отдельных продуктов.

| Комбинация | Что происходит |
|---|---|
| Fiber + Carb | Пик снижен ~30% (клетчатка замедляет всасывание) |
| Protein + Carb | Кривая сглажена и сдвинута (белок замедляет пищеварение) |
| Carb + Carb | Пик усилен нелинейно ×1.3 (перегрузка) |
| Fat + Sugar | Пик отложен, но хвост удлинён ×1.8 ("эффект мороженого") |

**Почему это интересно:** Нельзя оценить продукт изолированно. Банан — хороший или плохой? Зависит от соседей на тарелке. Рис + курица + салат → мягкий холм. Рис + банан → взрывной пик. Одно и то же решение о рисе, два противоположных результата. Игрок **экспериментирует**, а не считает.

**Почему это даёт вариативность:** 5 тегов × комбинации = десятки взаимодействий. Каждый новый продукт создаёт новое пространство комбинаций. При этом правила просты — сложность возникает из комбинаторики, а не из количества правил.

### 2. Межтарелочные зависимости — "завтрак ломает ужин"

Тарелки не независимы. BG-кривая от одного приёма может не успеть снизиться к следующему. Хвост завтрака поднимает основание обеда. Fat особенно коварен — откладывает пик, создавая **отложенное наложение**.

```
Тяжёлый завтрак → обед стартует с BG=140 вместо 100:

        ╱‾╲ ← обед уже в жёлтой зоне
  ╱‾╲  ╱   ╲
 ╱   ╲╱     ╲
╱     ↑       ╲
   хвост завтрака
```

**Почему это интересно:** Оптимизация одной тарелки бессмысленна без учёта остальных. Три "неплохих" тарелки могут давать катастрофический день, а одна "плохая" тарелка в правильном месте — допустимый результат. Это превращает задачу из 3 отдельных решений в одно глобальное.

**Почему это даёт вариативность:** Одна и та же тарелка даёт разный результат в зависимости от позиции в дне. Мороженое на ужин (после которого ничего нет) безопаснее, чем на обед (хвост наслаивается на ужин). Это неочевидный инсайт, который игрок открывает сам.

### 3. Дефицит рынка — "нет идеального решения"

Каждый день — ограниченный набор продуктов. Идеальная тарелка (Carb + Protein + Fiber) не всегда собираема: курица одна, а тарелок три. Кто-то останется без белка.

**Почему это интересно:** Дефицит заставляет выбирать. Не "что оптимально?", а "чем я готов пожертвовать?". Курица на завтрак (спокойное утро, но тяжёлый обед) или на обед (тяжёлое утро, но компенсированный обед)? Нет правильного ответа — есть trade-off.

**Почему это даёт вариативность:** Другой набор на рынке = другой пазл. 8 продуктов из пула в 20+ → тысячи комбинаций уровней без написания нового кода. При этом каждый набор "решаем", но по-разному.

### 4. Живой график — "видишь последствия до того, как совершил ошибку"

BG-кривая обновляется в реальном времени при каждом перетаскивании. Не числа в таблице — а форма горки, которая растёт или сглаживается.

**Почему это интересно:** Мгновенная визуальная обратная связь создаёт цикл "попробовал → увидел → понял → попробовал иначе". Игрок учится через эксперимент, а не через чтение правил. Добавил салат → увидел, как горка присела. Убрал курицу → увидел, как горка вытянулась. Причинно-следственная связь ощущается тактильно.

**Почему это даёт вариативность:** График показывает то, что нельзя вычислить в уме — наложение хвостов, нелинейные синергии, влияние порядка. Игрок вынужден экспериментировать, потому что результат непредсказуем интуитивно, но понятен визуально.

### 5. Деградация тела — "вчерашняя стратегия не работает сегодня"

По мере прогрессии игры тело слабеет (порог инсулинового ответа растёт). Тарелка, которая давала пик 180 на уровне 2, даёт пик 220 на уровне 6.

**Почему это интересно:** Нет "решённой" стратегии. Каждый уровень деградации вынуждает пересматривать подход. То, что работало вчера, ломается сегодня — игрок ищет новые комбинации в рамках тех же правил.

**Почему это даёт вариативность:** Одни и те же продукты → разные результаты → разные оптимальные тарелки. Деградация — бесплатный множитель вариативности: N продуктов × M уровней деградации = N×M разных пазлов.

### 6. Нарративные сценарии — "контекст меняет задачу"

Уровни обрамлены бытовыми ситуациями: бизнес-ланч (обед фиксирован), день рождения (торт обязателен), командировка (только фастфуд), болезнь (метаболизм замедлен).

**Почему это интересно:** Те же правила, но другая начальная позиция. Игрок не учит новые механики — он применяет изученные в новом контексте. Это ощущается как "жизненная ситуация", а не "уровень с цифрой 7".

**Почему это даёт вариативность:** Каждый сценарий — уникальная комбинация ограничений из уже существующих элементов. Десятки сценариев создаются комбинаторно: (фиксированный приём) × (набор рынка) × (уровень деградации) × (доступность упражнений).

## Прогрессия — введение по одной идее за уровень

| Уровень | Новая идея | Что игрок открывает |
|---|---|---|
| 1 | Одна тарелка, 3 продукта | Carb+Carb = плохо, Protein+Carb = буфер |
| 2 | Три тарелки, распределение | Тарелки влияют друг на друга через BG-хвосты |
| 3 | Клетчатка | Fiber+Carb = снижение пика (первая "зелёная" синергия) |
| 4 | Бесплатная вредная еда | Fat откладывает пик, но удлиняет хвост; бесплатное ≠ безвредное |
| 5 | Фиксированный приём | Компенсация "плохого" приёма соседними тарелками |
| 6 | Деградация тела | Адаптация стратегии к ослабленному метаболизму |
| 7 | Упражнение | Активность между приёмами "гасит" BG-кривую |
| 8 | Обязательный "плохой" продукт | Даже вредная еда управляема в правильном контексте |
| 9 | Порядок блюд | Овощи первыми → пик ниже (реальная клиническая рекомендация) |
| 10+ | Нарративные сценарии | Все инструменты в новых комбинациях ограничений |

## Связь с T2D — не метафора, а прямое отражение

Каждая механика соответствует реальной рекомендации для людей с диабетом 2-го типа:

| Механика | Реальная рекомендация |
|---|---|
| Fiber + Carb синергия | "Ешьте овощи с каждым приёмом пищи" |
| Protein + Carb синергия | "Сочетайте белок с углеводами" |
| Fat как ловушка | "Жирная пища не поднимает сахар сразу, но эффект длится дольше" |
| Межтарелочные хвосты | "Распределяйте приёмы пищи равномерно по дню" |
| Упражнение после еды | "Прогулка после еды снижает уровень сахара" |
| Порядок блюд | "Ешьте овощи и белок перед углеводами" |
| Деградация | "Без контроля чувствительность к инсулину снижается" |
| Дефицит рынка / бюджет | "Здоровое питание требует осознанного выбора" |

Игрок не заучивает правила — он **открывает** их через эксперимент с тарелками. Образовательный эффект возникает естественно, как побочный продукт решения пазла.
